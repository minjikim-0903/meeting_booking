# 미팅 예약 시스템 기획서

## 1. 배경 및 문제 정의

- 6~8명(최대 8명 기준 설계, 최소 6명)이 **다음 주 평일 5일** 내로 미팅을 잡아야 함
- 참여자는 4개 직무로 구성: **프로덕트 오너, 개발자, 데이터 분석가, 디자이너**
- 전사 **유연근무제** 적용 중 (출근 08:00~10:00, 퇴근 17:00~19:00, 야근 가능성 존재) → 개인별 근무 시간대가 서로 다름
- 각자 다음을 모두 고려해서 시간을 맞춰야 함
  - 개인 일정 (휴가, 개인 용무 등)
  - 이미 잡혀 있는 다른 회의 일정
  - 갑작스러운 병가 / 당일 연차·반차 발생 가능성
- 미팅을 주도(소집)하는 사람이 고정되어 있지 않음 → **누구나 미팅 소집자가 될 수 있어야 함**
- 결석자를 위한 대안(줌 참여, 회의록 요약 제공)이 시스템에 포함되어야 함

## 2. 목적

참여자들의 근무시간·개인일정·회의실 가용 여부를 한 화면에서 조율하고, 초대 → 사전 참석여부 체크 → 자동 확정(회의시간+회의실)까지 이어지는 예약 플로우를 제공한다.

## 3. 핵심 가설 및 조율 플로우

```
① 6~8명 미팅이 필요함
        ↓
② 가능한 회의실 확인 (요일 x 시간대별 회의실 예약 현황)  ← 우선순위 1
        ↓
③ 해당 회의실이 비어있는 시간대 확인
        ↓
④ 참여자 초대 시 각자의 일정(근무시간/기존 미팅/개인일정) 확인  ← 우선순위 2
        ↓
⑤ 참여자에게 "참여 가능 여부" 사전 검토 알림 발송
        ↓
⑥ 참여자가 이모티콘으로 참석/불참 응답
        ↓
⑦ 전원 "참석 가능" 시 → 회의시간 + 회의실 자동 예약 확정
   일부 불참/일정 충돌 시 → 시간 재조율 필요 (④로 회귀)
```

이 순서에 따라 **회의실 가용 현황 페이지를 최우선으로 구현**하고, 이후 개인 일정 확인 페이지를 붙인다.

## 4. 사용자 및 역할

| 역할 | 설명 |
|---|---|
| 참여자 (Participant) | 프로덕트 오너 / 개발자 / 데이터 분석가 / 디자이너. 본인 일정을 등록하고, 초대에 이모티콘으로 응답 |
| 소집자 (Organizer) | 미팅 초대를 발송하는 사람. 고정 역할이 아니며 참여자 누구나 될 수 있음 |
| (선택) 타팀 참여자 | 다른 팀 소속으로 별도 조율/양해 멘트가 필요한 참여자 |

## 5. 페이지 구성 및 기능 요구사항

### 5.1 회의실 가용 현황 페이지 (최우선 구현)

- 요일(월~금) x 시간대(**10분 단위**) 매트릭스로 회의실별 예약 가능 여부를 시각화
- 회의실별 예약 캘린더 뷰 제공 (회의실 선택 → 해당 회의실의 다음 주 예약 현황 확인)
- 회의실 정보: 수용 인원(최소 6~8명 수용 가능한 곳 필터링), 위치, 화상회의 장비 유무
- 빈 슬롯 클릭 시 바로 예약 플로우(5.4)로 진입 가능

### 5.2 개인 일정 등록 페이지

- 각 참여자가 본인 계정으로 로그인 후 본인 일정을 직접 등록
- 등록 항목
  - 근무 가능 시간대 (유연근무제 반영: 출근 08:00~10:00 중 선택, 퇴근 시간 자동 계산 또는 별도 입력, 야근 가능 여부)
  - 개인 일정 (휴가/외부 미팅/개인 용무 등 비공개 처리 가능, 시간대만 "불가"로 표시)
  - 기존 확정된 다른 미팅 일정 (연동 또는 수기 등록)
- 상태값 예시: 근무 가능 / 다른 미팅 / 개인 일정(비공개) / 연차 / 반차 / 병가
- 반차/연차/병가는 **당일 갑작스러운 변경**이 가능해야 하므로, 등록 즉시 관련된 예정 미팅에 알림이 가도록 처리 (5.6 참고)

### 5.3 통합 일정 조율 페이지 (참여자 스케줄 통합 뷰)

- 초대 예정/초대된 참여자 전원의 일정을 한 화면에 겹쳐서(타임라인/그리드) 표시
- 전원이 동시에 가능한 시간대(교집합)를 자동 하이라이트
- 회의실 가용 시간(5.1)과 교집합을 낸 "회의실 O + 전원 가능" 슬롯을 우선 추천
- 타 팀 참여자가 포함된 경우, 해당 인원의 일정은 별도 소스(공유 캘린더 연동 또는 수기 입력)로 표시하고 조율 난이도가 있음을 UI로 구분

### 5.4 미팅 예약(조율) 페이지

예약 시 확인/입력해야 하는 항목:

1. 회의실 선택 (달력 뷰로 해당 회의실의 가능 날짜 확인)
2. 회의 시간 확정 (5.3의 추천 슬롯 기반, **10분 단위로 시작/종료 시간 선택**)
3. 참여자 초대 (직무별 최소 6명 ~ 최대 8명 선택, 참여자 일정 자동 조회)
4. 타 팀과의 회의 조율이 필요한 경우
   - 타 팀 참여자 초대
   - 소집자가 **양해 멘트**를 작성해 함께 발송 (예: "타 팀 일정 우선 고려 부탁드립니다" 등 커스텀 메시지)
5. 화상회의 링크 자동 생성 (Google Meet 또는 Zoom 연동)
6. 예약 요청 발송 → 사전 검토 단계(5.5)로 이동

### 5.5 사전 참석 여부 검토 (초대 응답)

- 초대받은 참여자에게 알림 발송 (인앱 알림 + 이메일/슬랙 등 외부 채널 연동 고려)
- 참여자는 이모티콘 1탭으로 응답
  - ✅ 참석 가능
  - ❌ 참석 불가
  - ❓ 미정 (일정 확인 후 재응답)
- 응답 현황을 소집자가 실시간으로 확인 가능 (예: 5/6명 응답, 4명 가능)
- **전원 ✅ 응답 시 자동으로 회의시간 + 회의실 예약 확정** (5.6)
- 응답 마감 시한(예: 초대 후 24시간) 설정, 미응답 시 리마인드 알림

### 5.6 예약 확정 및 예외 처리

- 전원 참석 가능 확정 시
  - 회의실 자동 예약 확정
  - 화상회의 링크 확정 및 참여자 전원에게 캘린더 초대 발송
- 일부 불참/충돌 발생 시
  - 자동으로 대안 시간 슬롯 재추천 (5.3 로직 재실행)
  - 소집자에게 재조율 필요 알림
- 확정 이후 갑작스러운 변경 발생 시 (병가/당일 연차·반차)
  - 해당 참여자가 상태 변경 시 소집자 및 다른 참여자에게 즉시 알림
  - 결석자를 위한 대체 참여 옵션 제공
    - 화상회의(Zoom/Meet) 링크로 비동기 참여 유도
    - 회의 종료 후 회의록/요약 자동 공유 대상에 자동 포함

### 5.7 회의록 작성 및 요약 페이지

- 회의 중에는 수기로 회의록 작성 가능 (실시간 메모)
- **회의 종료 시점에 회의록 및 요약본이 자동 저장** (별도 저장 액션 불필요, 회의 상태가 "종료"로 전환되는 순간 트리거)
- 결석자를 고려한 **요약본 자동 생성 및 공유** 기능
- 회의록에 다음 포함
  - 참석자 / 불참자 목록
  - 논의 내용
  - 액션 아이템 (담당자, 기한)
- 회의 종료(자동저장) 즉시 결석자에게 회의록 링크 자동 발송

## 6. 데이터 모델 (초안)

- **User**: id, 이름, 직무(PO/개발자/데이터분석가/디자이너), 소속 팀
- **Schedule**: user_id, 날짜, 시작~종료시간, 상태(근무가능/다른미팅/개인일정/연차/반차/병가), 비공개 여부
- **MeetingRoom**: id, 이름, 수용인원, 위치, 화상장비 유무
- **RoomBooking**: room_id, 날짜, 시작~종료시간, meeting_id
- **Meeting**: id, 소집자_id, 제목, 상태(초안/검토중/확정/취소), 화상회의 링크, 양해_멘트
- **MeetingInvite**: meeting_id, user_id, 응답상태(가능/불가/미정), 응답시각
- **MeetingNote**: meeting_id, 작성자_id, 내용, 요약본, 액션아이템, 자동저장_시각(회의 종료 시각과 동일)

## 7. 구현 우선순위 (MVP 로드맵)

| 단계 | 범위 |
|---|---|
| MVP 1 | 회의실 가용 현황 페이지 (요일 x 시간대 매트릭스) |
| MVP 2 | 개인 일정 등록 페이지 + 통합 일정 조율 페이지(교집합 뷰) |
| MVP 3 | 미팅 예약 페이지 (회의실+시간 확정, 참여자 초대, 화상회의 링크 연동) |
| MVP 4 | 사전 참석 여부 이모티콘 응답 + 전원 동의 시 자동 확정 |
| MVP 5 | 예외 처리(당일 연차/반차/병가 알림), 회의록/요약 페이지, 타 팀 조율(양해 멘트) |

## 8. 문제 발견 및 개선 방향 (설계 노트)

기획자가 가설을 세우고 설계하면서 스스로 발견한 문제와, 그에 대한 개선 방향을 기록한 노트.

> 아래 8.1~8.4는 최초 작성 버전이며, 8.5에서 개정되었다. 8.5를 최신 기준으로 본다.

### 8.1 가설 (재정리, v1)

팀원의 스케줄과 미팅룸 시간을 확인할 수 있게 하고, 예상치 못한 연차·재택근무 발생 시에는 회의록 요약 공유와 화상미팅으로 참석을 대체할 수 있게 한다.

**1. 팀의 회의를 잡기 위해서 필요한 것**
- 미팅룸 확인
- 각자의 스케줄 확인
- 미팅 가능 시간 확인

**2. 고려해야 할 것**
- 예상치 못한 연차 사용
- 팀원의 타 미팅 시간
- 비어있는 회의룸
- 예약이 되더라도 예상치 못한 미참석 대응
- 회의 목적에 따라 중요도에 따라 스케줄을 조정해야 함

### 8.2 문제 발견 (v1)

1. 미팅을 잡을 목적이라 해도 참석자의 디테일한 스케줄을 공개할 의무가 없다.
2. 미팅을 잡기 위해 팀원이 직접 스케줄을 수정/등록해야 한다.
3. 미팅 시간이 변경되거나 예약이 겹칠 경우 조율이 필요하다.

### 8.3 개선 방향 (v1)

1. **가능/불가능 여부로만 표기한다** — 참석자의 세부 일정 내용은 노출하지 않고, 상태값(가능/불가능)만 초대자에게 보여준다. (문제 1에 대한 대응 → 참석자 일정 확인 기능으로 구현)
2. **HR 시스템과 연동한다** — 연차/반차 등록을 팀원이 이 시스템에 이중으로 입력하지 않도록, 사내 HR 시스템과 연동해 자동 반영한다. (문제 2에 대한 대응, 미구현 — 9장 열린 질문 참고)
3. **조율 버튼 + 우선순위 파악** — 미팅 시간 변경/예약 충돌 시 "조율" 버튼을 통해 회의 목적에 따른 우선순위를 매기고, 중요도가 높은 회의를 우선 확정한다. (문제 3에 대한 대응, 미구현)
4. **회의 목적 우선순위 기준 필요** — 전사적으로 동의된 우선순위 판단 기준이 있어야 조율이 가능함. 화면 우측에 기준을 노출 (기준 항목은 우선 임시로 채워 넣고, 추후 전사 합의 기준으로 교체 예정).

### 8.4 로드맵 반영 (v1)

8.3의 1번(가능/불가능 표기)은 이미 구현 반영됨(참석자 일정 확인 패널). 2~4번은 아직 미구현 상태이며, MVP 로드맵(7장) 이후 단계로 별도 우선순위 조율 기능 추가가 필요함.

### 8.5 가설 · 문제 발견 · 개선 방향 (개정, v2)

**가설**: 팀원의 스케줄과 미팅룸 시간을 확인할 수 있게 하고, 예상치 못한 연차·재택근무 발생 시에는 회의록 요약 공유와 화상미팅으로 참석을 대체할 수 있게 한다.

**1. 팀의 회의를 잡기 위해서 필요한 것**
- 미팅룸 확인
- 각자의 스케줄 확인
- 미팅 가능 시간 확인

**2. 고려해야 할 것** (최종)
- 예상치 못한 연차 사용
- 팀원의 타 미팅 시간
- 비어있는 회의룸
- 예약이 되더라도 예상치 못한 미참석 대응
- 스케줄을 조정 및 조율이 되어야 한다
- 다른 팀/다른 팀원에서도 예약 일정을 확인할 수 있어야 한다
- **시간 제약 없이 어디서든 미팅 스케줄을 확인하고 등록/수정/삭제할 수 있어야 한다** (신규 — 반응형/모바일 접근성 요구. 웹 기반이므로 원칙적으로 충족되나, 별도 반응형 QA는 아직 미실시)

**문제 발견** (최종)

1. 회의실을 잡기 위해 예약 조율, 팀원 스케줄 파악에 드는 리소스(시간·수고)가 많이 든다.
2. 팀원의 스케줄과 미팅룸 예약을 한 페이지에 같이 보여주려 하니 정보 과부하가 생긴다.
3. 미팅을 잡을 목적이라 해도 참석자의 디테일한 스케줄을 공개할 의무가 없다.
4. 미팅을 잡기 위해 팀원이 직접 스케줄을 수정/등록해야 한다.
5. 미팅 시간이 변경되거나 예약이 겹칠 경우 조율이 필요하다.

**개선 방향** (최종)

> 회의실 예약과 개인 스케줄은 서로 다른 기능이므로, 화면에서 각각 별도 컬럼으로 구분한다. (문제 1·2에 대한 대응 → 회의실(좌) / 참석자(우) 2컬럼 레이아웃으로 구현 완료. 회의실 관련 필터는 좌측 회의실 컬럼에, 참석자/팀 필터는 우측에 별도로 배치)

1. **내 팀만 선택해도 회의실 예약 가능 여부를 파악할 수 있게 한다** (팀을 고르면 해당 팀원 전원이 자동으로 참석자 목록에 반영되도록 구현 완료)
2. **HR 시스템과 연동해서 재택, 연차, 외근을 표기한다** (미구현 — 9장 열린 질문 참고)
3. **조율 버튼 + 우선순위 파악** — 미팅 시간 변경/예약 충돌 시 "조율" 버튼을 통해 회의 목적에 따른 우선순위를 매기고, 중요도가 높은 회의를 우선 확정한다 (미구현)
4. **회의 목적 우선순위 기준 필요** — 전사적으로 동의된 우선순위 판단 기준이 있어야 조율이 가능함. 화면 우측에 기준을 노출 (기준 항목은 우선 임시로 채워 넣고, 추후 전사 합의 기준으로 교체 예정, 미구현)

### 8.6 로드맵 반영 (최종)

- 개선 1번은 구현 완료 (팀 선택 시 팀원 자동 반영). "가능/불가능 여부로만 표기"는 참석자 일정 확인 기능으로 이미 구현되어 있어 별도 개선 항목에서는 제외.
- 화면 레이아웃: 좌측 = 회의실(회의실 필터 포함) 컬럼, 우측 = 참석자/팀 선택(별도 필터) 컬럼으로 구현 완료.
- "다른 팀/다른 팀원 예약 확인" 요구는 로그인/세션이 없는 현재 MVP 특성상, 별도 "내 팀" 개념 없이 **예약을 클릭하면 팀·예약자·참석자(팀원)가 항상 보이는 방식**으로 단순화하여 구현 완료.
- HR 연동(재택/연차/외근 표기), 조율 버튼(우선순위), 회의 목적 우선순위 기준은 아직 미구현 — MVP 로드맵(7장) 이후 단계로 별도 진행 필요.
